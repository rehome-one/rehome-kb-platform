# Architecture Decision Records (ADR)

Этот каталог содержит записи архитектурных решений проекта reHome.

## Что такое ADR

ADR — короткий документ, фиксирующий значимое архитектурное решение:
- Какая проблема решалась
- Какое решение принято
- Какие альтернативы рассматривались и почему отвергнуты
- Какие последствия и компромиссы

ADR — иммутабельный документ. Если решение пересматривается, создаётся новый ADR
со ссылкой «Заменяет ADR-NNNN».

## Когда писать ADR

- Решение влияет на ≥ 2 модуля проекта.
- Решение трудно или дорого изменить позже.
- Решение принимается в спорной ситуации (с альтернативами).
- При запросе на отступление от правил из ТЗ.

## Формат

Используйте `0000-template.md` как основу. Имя файла: `NNNN-краткое-название.md`
(четырёхзначный порядковый номер, lower-kebab-case).

## Жизненный цикл

1. **Предложено** — кто-то предложил решение, обсуждается.
2. **Принято** — Архитектор подтвердил, решение в силе.
3. **Заменено ADR-MMMM** — старое решение отменено новым ADR.
4. **Отклонено** — решение рассматривалось и отвергнуто (сохраняется для истории).

## Правила

- ADR ссылается на конкретные пункты ПЗ/ТЗ.
- Автор ADR — обычно Архитектор или Tech Lead.
- Согласовано Архитектором — обязательно для статуса «Принято».
- Никогда не удаляйте принятые ADR — только заменяйте новыми.
